​​Новости и статьи о Quarkus за вторую половину июня

Самой главной новостью конечно же является релиз Quarkus 2.0! Хотя разработчики заявляют, что этот релиз является скорее последовательным развитием фреймворка и не включает каких-то революционных изменений, но тем не менее изменений довольно много, вот основные из них:
• Переход на Vert.x 4. Это само по себе хорошо, т.к. новая версия Vert.x предоставляет ряд новых возможностей, таких как - использование future вместо callback, поддержка Opentracing и Zipkin в Vert.x Tracing, реактивные SQL и Redis клиенты и множество других. Подробней можно почитать в оффициальном блоге Vert.x
• Переход на Eclipse MicroProfile 4. Недавний релиз MicroProfile 4.0 тоже принёс множество изменений во всех 12 основных API. Подробней можно почитать в статье на InfoQ
• Continuous Testing - новый инструмент, который позволяет ускорить разработку - он перезапускает юнит тесты для кода, над которым вы в данный момент работаете, и вы можете получать мнговенный фидбэк о ваших изменениях
• Quarkus CLI - инструмент коммандной строки, который позволяет вам создавать Quarkus проекты, управлять расширениями и собирать/запускать ваши Quarkus приложения. Подробней можно почитать в статье
• Добавлен GraphQL клиент, доступный в виде расширения для клиента SmallRye GraphQL
• Hibernate ORM был обновлён до версии 5.5
• ArC, реализация инъекции зависимостей и контекстов (CDI), теперь поддерживает CDI декораторы
• MongoDB с использованием Panache теперь поддерживает транзакции
• Улучшена поддержка Kotlin - расширения RESTEasy Reactive, REST Client и Reactive Messaging теперь поддерживают корутины
• Добавлено расширение для Amazon SSM
• JDK 11 вместо JDK 8 как минимальная поддерживаемая версия в JVM режиме
• GraalVM 21.1 как минимальная поддерживаемая версия

Подробнее о всех изменениях релиза можно почитать на оффициальном блоге, а также доступна запись последнего подкаста на Youtube, где разработчики рассказывали о релизе.

Новости Quarkus за вторую половину июля

Вторая половина июля порадовала нас двумя релизами Quarkus. Первый - Quarkus 2.0.3.Final - очередной багфикс релиз с исправлениями проблем, обновлением версий некоторых зависимостей и улучшением документации. Полный список изменений доступен на гитхабе.

Второй релиз более значительный - вышел Quarkus 2.1.0.Final. Среди основных изменений заявлены:
• Dev Service для Keycloak - если вы используете quarkus-oidc расширение и запускаете свой сервис в dev моде, автоматически будет стартовать Keycloak контейнер. Это позволит не тратить время на ручную конфигурацию и ускорит разработку, отладку и тестирование вашей функциональности. Keycloak - инструмент для реализации single sign-on и управления аутентификацией и авторизацией;
• расширение с реактивным драйвером для Microsoft SQL Server - теперь и с этой базой данных можно работать в реактивном стиле;
• поддержка Kotlin 1.5 - как ранее заявляли разработчики, работа над поддержкой языка Kotlin ведётся, хотя она и отстаёт от Java, на которой сконцентрированы основные силы;
• также в релиз вошло значительное количество различных исправлений и улучшений, полный список изменений доступен на гитхабе.

• Kubernetes-native Java microservices with John Clingan - эпизод подкаста "Semaphore Uncut" с Джоном Клиганом (John Clingan), продукт менеджером Red Hat, основателем MicroProfile и соавтором книги "Kubernetes Native Microservices with Quarkus, and MicroProfile". Новости Quarkus за вторую половину августа

С момента выхода нашего предыдущего поста в середине августа успело выйти два багфикс релиза для Quarkus 2.1 - 2.1.3.Final и 2.1.4.Final.

Также вышел релиз Quarkus 2.2.1.Final. Вы можете спросить, почему сразу 2.2.1 и где 2.2.0? Из-за бага не позволяющего запускать режим резработки (dev mode) на Windows были выпущены не все артифакты новой версии (только артифакты ядра, но не всей платформы). В итоге разработчики решили сконцентрироваться на стабилизации версии 2.2 в целом - работать над испрвлением багов, улучшать юзабилити и документацию.

Таким образом, список заслуживающих внимания новых функций в этом выпуске невелик:
- обновление до GraalVM 21.2
- добавление глобального флага для отключения режима разработки (dev mode)
- изменение модели потоков по умолчанию для RESTEasy Reactive
- добавление поддержки MongoDB в сервисах разработки (DevServices)
- расширение для запуска Narayana LRA participants
